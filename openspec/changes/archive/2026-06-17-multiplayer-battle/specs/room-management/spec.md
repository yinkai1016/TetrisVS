## ADDED Requirements

### Requirement: 创建房间
系统 SHALL 允许玩家创建房间，创建时选择人数上限（2/3/4）。服务器生成唯一的 6 位纯数字房间 ID（`000000`–`999999`，查重）并返回给创建者。

#### Scenario: 创建房间得到 6 位数字 ID
- **WHEN** 玩家选择人数上限并创建房间
- **THEN** 服务器返回唯一的 6 位纯数字房间 ID，创建者进入该房间等待

#### Scenario: 房间 ID 唯一
- **WHEN** 服务器生成的新 ID 与现存房间 ID 冲突
- **THEN** 服务器重新生成直到不冲突

### Requirement: 加入房间
系统 SHALL 允许玩家输入 6 位房间 ID 加入对应房间。ID 不存在或房间已满时拒绝并提示。

#### Scenario: 正确 ID 加入成功
- **WHEN** 玩家输入有效且未满的房间 ID
- **THEN** 玩家加入该房间，房间内所有玩家收到人员更新

#### Scenario: 错误 ID 被拒
- **WHEN** 玩家输入不存在或已满的房间 ID
- **THEN** 加入失败，向玩家提示原因

### Requirement: 房间状态与人员同步
系统 SHALL 向房间内所有玩家同步当前人员列表与房间配置（人数上限、房主）。

#### Scenario: 人员变动同步
- **WHEN** 有玩家加入或离开房间
- **THEN** 房间内所有玩家收到更新后的成员列表

### Requirement: 房主开始对局
系统 SHALL 仅允许房主发起开始对局；开始时房间人数须 ≥2。

#### Scenario: 房主开始
- **WHEN** 房主在人数 ≥2 时点击开始
- **THEN** 服务器触发对局开局流程（见 multiplayer-session）

#### Scenario: 人数不足无法开始
- **WHEN** 房主在房间仅自己时点击开始
- **THEN** 开始被拒绝并提示需要更多玩家

### Requirement: 离开房间
系统 SHALL 允许玩家主动离开房间；离开后若房间无人则销毁房间，若房主离开则转移房主给剩余成员。

#### Scenario: 房主离开转移房主
- **WHEN** 房主离开且房间仍有其他成员
- **THEN** 房主权限转移给某位剩余成员
